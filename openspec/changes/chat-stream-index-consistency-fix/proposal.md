## Why

当前项目存在三个用户可见的一致性缺口：(1) 聊天流式输出在用户切换会话后仍可能继续写入错误的聊天视图；(2) 知识库索引进度在失败后可能卡在 UI 中无法恢复；(3) 同步知识库索引与流式索引未保持相同的持久化文件状态语义。这些问题破坏了 UI 状态的信任度，使用户难以判断哪个聊天或文件状态是权威的。

## What Changes

- **聊天流生命周期安全**：确保会话聊天页面在切换聊天或删除当前选中聊天时取消正在进行的流式响应，防止流式 token 泄漏到错误的本地聊天视图中。
- **知识库进度状态恢复**：强化前端 KB 索引进度处理，使文件级和批量索引指示器在成功、SSE 错误事件和请求级失败时都能干净地收敛到终态。
- **同步与流式索引状态一致**：使同步 KB 索引路径持久化与流式索引路径相同的文件状态契约，包括正确的 `indexed` 状态更新和文件级分块计数，而非集合级总数。
- **回归测试覆盖**：为聊天流取消行为、KB 索引失败状态清理以及同步/流式索引状态一致性增加回归测试覆盖。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `session-chat`：明确流式聊天输出必须限定在本地选中的目标聊天范围内，一旦该聊天不再是当前活动本地视图，必须停止 UI 更新。
- `ui-session-page`：强化本地聊天视图状态处理，使聊天切换和当前聊天删除能取消过时的流，而非让过时输出继续变更当前页面状态。
- `api-kb`：收紧 KB 索引响应语义，使同步索引和重新索引保持与流式索引一致的持久化文件状态元数据。
- `ui-kb-page`：要求索引进度指示器能在失败后干净恢复，并反映最终权威的文件状态。
- `auto-test`：增加覆盖流取消和 KB 索引状态一致性的回归测试要求，涵盖成功路径和失败路径。

## Impact

- 前端会话流程：`ui/src/pages/SessionChat.tsx`、相关聊天状态连接代码以及 `ui/src/api/index.ts` 中的流生命周期处理
- 前端 KB 流程：`ui/src/pages/KbDetail.tsx` 及相关的 SSE 进度处理
- 后端 KB 索引：`app/modules/kb_manager/indexer.py`、`app/api/routers/kb.py` 以及 `app/modules/kb_manager/knowledge_base.py` 中的文件状态持久化辅助函数
- 测试：`tests/unit` 中的新增项，以及可能对 API 可见索引语义的集成测试覆盖
