## Why

当前项目在三个关键边界上存在系统性风险：一是知识库与会话相关路径缺少根目录约束，外部输入可以越界到预期存储目录之外；二是混合检索中的 BM25 缓存只按向量数量失效，重建索引后可能继续混入旧文本；三是同步聊天与流式聊天的持久化格式不一致，导致前端历史回放与来源展示行为分叉。随着 Web、CLI、会话流式对话和知识库索引能力已经并存，这些问题已经从局部缺陷升级为数据安全和系统一致性问题，需要统一加固。

## What Changes

- 为知识库名、会话名、文件名、聊天文件名建立统一的路径约束与合法性校验，确保所有文件读写、删除、列举和自动创建行为都被限制在各自根目录内。
- 为上传、上传并索引、删除文件、删除聊天、读取聊天消息等 API/CLI 路径增加稳定的非法路径拒绝行为，并定义一致的错误响应。
- 调整混合检索缓存失效策略，使 BM25 侧在索引内容变化后不会仅因 chunk 数量未变而继续复用陈旧文本。
- 统一同步聊天与流式聊天的持久化协议，要求二者都以相同结构保存助手消息与来源信息。
- 调整会话详情与历史消息读取契约，使前端在重新加载聊天记录时能够稳定恢复结构化来源，而不是依赖流式期间的临时内存状态。
- 将前端会话页纳入同一变更，确保历史来源展示、错误处理和契约消费方式与新的后端持久化格式保持一致。

## Capabilities

### New Capabilities
- `storage-path-guardrails`: 定义知识库与会话存储路径的根目录约束、合法名称规则，以及越界输入的拒绝行为。

### Modified Capabilities
- `api-kb`: 调整知识库 API 的上传、上传并索引、删除与目标路径处理要求，要求所有文件目标都经过路径约束校验。
- `api-session`: 调整会话 API 的 `chat_file`、会话名和读取/删除聊天文件行为，要求使用统一的受限路径契约与稳定错误响应。
- `session-chat`: 调整检索与聊天持久化要求，要求重建索引后混合检索不使用陈旧 BM25 文本，并要求同步/流式聊天使用统一消息存储格式。
- `ui-session-page`: 调整会话页历史消息恢复与来源展示要求，要求从统一的结构化消息持久化中恢复来源信息。

## Impact

- 受影响后端代码：`app/modules/kb_manager/knowledge_base.py`、`app/modules/kb_manager/indexer.py`、`app/modules/retrieval/retriever.py`、`app/modules/session/session_manager.py`
- 受影响 API：`/api/kb/*`、`/api/session/*`
- 受影响前端页面与数据访问：`ui/src/pages/SessionChat.tsx`、`ui/src/components/ChatArea.tsx`、`ui/src/api/index.ts`
- 受影响测试：路径安全、重建索引后的检索一致性、同步/流式聊天记录一致性、前端历史来源恢复相关测试需要补充或更新
