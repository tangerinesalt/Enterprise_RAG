## Context

多页 PDF 文档的页眉页脚（文件编号、版本号、页码、文档标题等）会在每个 chunk 中重复出现。当前管线中：

```
PDF → RobustPDFReader (逐页 Document)
    → chunker.chunk_documents (SentenceSplitter)
        → nodes (每个 node 保留原始文本)
```

没有任何一步处理重复文本。对于 11 页的文档，同一段页眉会出现在 11 个 chunk 中，在嵌入空间产生语义坍缩。

### 当前管线的数据路径

```
                   页眉页脚传播路径
        
  第1页: [页眉] 目的与范围 [页脚]     → chunk_1: "[页眉] 目的与范围 [页脚]"
  第2页: [页眉] 职责分工 [页脚]       → chunk_2: "[页眉] 职责分工 [页脚]"
  第3页: [页眉] 管理流程 [页脚]       → chunk_3: "[页眉] 管理流程 [页脚]"
   ...                                → ...
  第N页: [页眉] 附则 [页脚]           → chunk_N: "[页眉] 附则 [页脚]"

嵌入空间中的效果:
  d(chunk_1, chunk_2) = 本该反映 "目的与范围 vs 职责分工"
                      但实际被页眉页脚的共享文本拉近
```

## Goals / Non-Goals

**Goals:**
- 在分块前检测并剥离跨页重复的页眉页脚文本
- 纯文本规则检测，不依赖外部模型或 API
- 算法与语言无关（不写死中文/英文关键词）
- 第一版实现只处理文档头尾的重复行，不处理页眉页脚出现在正文中间的复杂情况

**Non-Goals:**
- 不涉及 OCR 管线的修改
- 不做表格结构重建（属于单独的提案 2）
- 不做通用的文本清洗（控制字符、CJK 空格等属于后续优先级）
- 不修改现有的 SentenceSplitter 分块参数

## Decisions

### Decision 1：在 chunker.py 做预处理，而不是 RobustPDFReader

**选择**：在 `chunk_documents()` 中跨 Document 检测页眉页脚，在传进 SentenceSplitter 前清洗。

**理由**：
- `RobustPDFReader.load_data()` 返回逐页 Document，此时有完整的"全文档"视图可以计算跨页重复
- `chunker.py` 是管线的最后文本处理层，在此清洗后直接送入 Embedding，不会有其他环节重新引入噪声
- 对 pypdf 提取和 OCR 提取一视同仁——两种来源都经过 `chunk_documents()`
- 对比在 `RobustPDFReader` 里做：需要额外传递"全文档"上下文，破坏了逐页处理的简洁性

### Decision 2：检测策略——取每页首/末 N 行，统计跨页重复率

**算法**：

```
对于文档的每一页 p：
  取 p 的前 3 行 → heads[p]
  取 p 的后 3 行 → tails[p]

计算所有 heads 和 tails 中每一行出现的页数

对于每一行：
  如果出现页数 / 总页数 > 0.5 → 标记为"重复页面元素"

从每页文本中移除所有被标记的行
```

**理由**：
- 简单、可预测、O(n) 时间
- 0.5 阈值确保只去除真正"大多数页都有"的文本
- 同时覆盖页眉和页脚
- 行级匹配对换行敏感，最多误删 1-2 行，不会破坏正文

**风险**：如果某段正文巧合地出现在多数页的相同位置（如表格的多页连续性），可能被误删。但概率低，且可以通过后续验证检出。

### Decision 3：不做 fuzzy 匹配（不做文本相似度比较）

**选择**：精确的行字符串比较。

**理由**：
- 页眉页脚通常是精确重复的（同一文件编号、同一版本号）
- 如果真的换了（比如奇偶页页眉不同），精确匹配也不会误杀
- Fuzzy 匹配增加了复杂度（需要选阈值、处理误报），收益不明确

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|---------|
| 跨页重复的正文被误判为页眉 | 阈值设为 0.5（超过一半的页出现才判） |
| OCR 文本中页眉页脚本身有 OCR 噪声，导致精确匹配失败 | 如果此情况普遍，可以降级加 fuzzy 匹配，但先不做 |
| 文档只有 1-2 页，重复率统计不可靠 | 总页数 < 3 时跳过检测 |
| 去重后 chunk 的上下文完整性受影响（如页脚包含关键引用信息） | 页脚通常只包含页码和文件编号，正文引用的关键信息不应只出现在页脚 |
